# 快速开始

import Tabs from "@theme/Tabs";
import TabItem from "@theme/TabItem";
import APITable from "@site/src/components/APITable";

RAGFlow 是一个基于深度文档理解的开源 RAG（检索增强生成）引擎。与 LLM 集成后，它能够提供可信的问答能力，并附带来自各种复杂格式数据的可靠引用。

本快速开始指南概述了从以下步骤的通用流程：

- 启动本地 RAGFlow 服务器，
- 创建数据集，
- 干预文件解析，
- 基于您的数据集建立 AI 对话。

:::danger 重要提示
我们官方支持 x86 CPU 和 Nvidia GPU，本文档提供在 x86 平台上使用 Docker 部署 RAGFlow 的说明。虽然我们也在 ARM64 平台上测试 RAGFlow，但不维护 RAGFlow 的 ARM Docker 镜像。

如果您使用 ARM 平台，请按照[本指南](./develop/build_docker_image.mdx)构建 RAGFlow Docker 镜像。
:::

## 前置条件

- CPU ≥ 4 核（x86）；
- 内存 ≥ 16 GB；
- 磁盘 ≥ 50 GB；
- Docker ≥ 24.0.0 且 Docker Compose ≥ v2.26.1。
- [gVisor](https://gvisor.dev/docs/user_guide/install/)：仅当您打算使用 RAGFlow 的代码执行器（[sandbox](https://github.com/infiniflow/ragflow/tree/main/sandbox)）功能时才需要安装。

:::tip 提示
如果您尚未在本地机器（Windows、Mac 或 Linux）上安装 Docker，请参阅 [安装 Docker Engine](https://docs.docker.com/engine/install/)。
:::

## 启动服务器

本节提供在 Linux 上设置 RAGFlow 服务器的说明。如果您使用的是其他操作系统，不用担心，大部分步骤都是类似的。

1. 确保 `vm.max_map_count` ≥ 262144。

<details>
  <summary>展开以显示详情：</summary>

`vm.max_map_count`。此值用于设置进程可拥有的最大内存映射区域数。其默认值为 65530。虽然大多数应用程序需要的映射数少于一千，但减小此值可能导致异常行为，且当进程达到限制时系统会抛出内存不足错误。

RAGFlow v0.24.0 使用 Elasticsearch 或 [Infinity](https://github.com/infiniflow/infinity) 进行多次召回。正确设置 `vm.max_map_count` 的值对于 Elasticsearch 组件的正常运行至关重要。

<Tabs
  defaultValue="linux"
  values={[
    {label: 'Linux', value: 'linux'},
    {label: 'macOS', value: 'macos'},
    {label: 'Windows', value: 'windows'},
  ]}>
  <TabItem value="linux">
   1.1. 检查 `vm.max_map_count` 的值：

```bash
$ sysctl vm.max_map_count
```

1.2. 如果未设置为 262144，则重置 `vm.max_map_count`：

```bash
$ sudo sysctl -w vm.max_map_count=262144
```

:::caution 警告
此更改将在系统重启后重置。如果您忘记在下一次启动服务器时更新该值，可能会收到 `Can't connect to ES cluster` 异常。
:::

1.3. 要使更改永久生效，请相应地在 **/etc/sysctl.conf** 中添加或更新 `vm.max_map_count` 值：

```bash
vm.max_map_count=262144
```

  </TabItem>
  <TabItem value="macos">
   如果您使用 macOS 上的 Docker Desktop，请运行以下命令来更新 `vm.max_map_count`：

```bash
docker run --rm --privileged --pid=host alpine sysctl -w vm.max_map_count=262144
```

:::caution 警告
此更改将在系统重启后重置。如果您忘记在下一次启动服务器时更新该值，可能会收到 `Can't connect to ES cluster` 异常。
:::

要使更改永久生效，请创建具有适当设置的文件：

1.1. 创建文件：

```shell
sudo nano /Library/LaunchDaemons/com.user.vmmaxmap.plist
```

1.2. 打开文件：

```shell
sudo launchctl load /Library/LaunchDaemons/com.user.vmmaxmap.plist
```

1.3. 添加设置：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.user.vmmaxmap</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/sbin/sysctl</string>
        <string>-w</string>
        <string>vm.max_map_count=262144</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
</dict>
</plist>
```

1.4. 保存文件后，加载新守护进程：

```shell
sudo launchctl load /Library/LaunchDaemons/com.user.vmmaxmap.plist
```

:::note
如果上述步骤无效，可考虑使用此解决方法，该方法使用容器，无需手动编辑 macOS 设置。
:::

  </TabItem>
  <TabItem value="windows">

#### 如果您在使用 Docker Desktop 的 Windows 上，则必须使用 docker-machine 来设置 `vm.max_map_count`：

```bash
$ docker-machine ssh
$ sudo sysctl -w vm.max_map_count=262144
```

#### 如果您在使用 Docker Desktop WSL 2 后端的 Windows 上，请使用 docker-desktop 来设置 `vm.max_map_count`：

1.1. 在 WSL 中运行：

```bash
$ wsl -d docker-desktop -u root
$ sysctl -w vm.max_map_count=262144
```

:::caution 警告
此更改将在您重启 Docker 后重置。如果您忘记在下一次启动服务器时更新该值，可能会收到 `Can't connect to ES cluster` 异常。
:::

1.2. 如果您更喜欢在每次重启 Docker 时不运行这些命令，可按以下方式更新您的 `%USERPROFILE%.wslconfig` 以使更改永久保持并全局应用于所有 WSL 发行版：

```bash
[wsl2]
kernelCommandLine = "sysctl.vm.max_map_count=262144"
```

_这会导致所有 WSL2 虚拟机在启动时分配该设置。_

:::note
如果您使用的是 Windows 11 或 Windows 10 版本 22H2，并已安装 Microsoft Store 版本的 WSL，您还可以在 docker-desktop WSL 发行版中更新 **/etc/sysctl.conf** 以使更改永久保持：

```bash
$ wsl -d docker-desktop -u root
$ vi /etc/sysctl.conf
```

```bash
# 添加一行，内容为：
vm.max_map_count = 262144
```

:::

## 配置 LLM

RAGFlow 是一个 RAG 引擎，需要与 LLM 配合才能提供可靠的、无幻觉的问答能力。RAGFlow 支持主流的 LLM。如需查看支持模型的完整列表，请参阅 [支持的模型](./guides/models/supported_models.mdx)。

:::note
RAGFlow 还支持使用 Ollama、Xinference 或 LocalAI 在本地部署 LLM，但这部分内容不在本快速入门指南的覆盖范围内。
:::

要添加和配置 LLM：

1. 点击页面右上角的您的头像 **>** **模型供应商**。

2. 点击所需的 LLM 并相应更新 API 密钥。

3. 点击 **系统模型设置** 以选择默认模型：
   - 聊天模型，
   - 嵌入模型，
   - 图像转文本模型，
   - 等等。

> 某些模型（如图像转文本模型 **qwen-vl-max**）隶属于特定的 LLM。您可能需要更新 API 密钥才能访问这些模型。

## 创建您的第一个数据集

您可以向 RAGFlow 中的数据集上传文件，并将其解析为数据集。数据集本质上是多个数据集的集合。RAGFlow 中的问答可以基于特定数据集或多个数据集。RAGFlow 支持的文件格式包括文档（PDF、DOC、DOCX、TXT、MD、MDX）、表格（CSV、XLSX、XLS）、图片（JPEG、JPG、PNG、TIF、GIF）和幻灯片（PPT、PPTX）。

创建您的第一个数据集：

1. 点击页面顶部中间的 **数据集** 选项卡 **>** **创建数据集**。

2. 输入您的数据集名称，点击 **OK** 确认更改。

   _您将进入数据集的 **配置** 页面。_

   ![dataset configuration](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/configure_knowledge_base.jpg)

3. RAGFlow 提供多种分块模板，以满足不同的文档布局和文件格式要求。为您的数据集选择嵌入模型和分块方法（模板）。

:::danger IMPORTANT
一旦您选择了嵌入模型并用它解析了文件，您将不再被允许更改它。原因很明显：我们必须确保特定数据集中的所有文件都使用**相同的**嵌入模型进行解析（确保它们在同一个嵌入空间中进行比较）。

# 快速开始

本指南将帮助你了解 RAGFlow 的核心功能：从创建知识库、上传文档、解析文档，到创建 AI 对话。

## 创建知识库

1. 点击页面顶部的 **Knowledgebase** **>** **Create knowledgebase** 创建知识库：

   ![create kb](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/create_kb.png)

2. 为你的知识库命名并添加描述（可选）。

3. 如果需要，可以为知识库配置不同的嵌入模型和 Rerank 模型。
   - **Embedding model**: 将文本转换为向量表示。
   - **Rerank model**: 对初步检索结果进行重新排序，以提高准确性。

4. 确认后，你将进入知识库的 **Dataset** 页面。

## 上传文档

1. 创建知识库后，你将被带往知识库的 **Dataset** 页面。

2. 点击 **+ Add file** **>** **Local files** 开始向知识库上传特定文件。

3. 在上传的文件条目中，点击播放按钮开始解析文件：

   ![parse file](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/parse_file.jpg)

   :::caution 注意
   - 如果你的文件解析在 1% 以下卡住，请参阅[此常见问题解答](./faq.mdx#为什么我的文档解析在百分之一以下卡住)。
   - 如果你的文件解析在接近完成时卡住，请参阅[此常见问题解答](./faq.mdx#为什么我的-pdf-解析在接近完成时卡住而日志没有显示任何错误)
     :::

## 干预文件解析

RAGFlow 具有可视性和可解释性，让你能够查看分块结果并在必要时进行干预。具体操作如下：

1. 点击完成解析的文件，查看分块结果：

   你将进入 **Chunk** 页面：

   ![chunks](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/file_chunks.jpg)

2. 将鼠标悬停在每个快照上，可以快速查看每个块的内容。

3. 双击分块后的文本，添加关键词或进行*手动*修改：

   ![update chunk](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/add_keyword_question.jpg)

   :::caution 注意
   你可以为文件块添加关键词或问题，以提高包含这些关键词的查询的排名。此操作会增加其关键词权重，并可提高其在搜索列表中的位置。
   :::

4. 在检索测试中，在 **Test text** 中输入快速问题，以检查你的配置是否有效：

   从下图可以看出，RAGFlow 返回了真实可靠的引用。

   ![retrieval test](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/retrieval_test.jpg)

## 设置 AI 对话

RAGFlow 中的对话基于特定的知识库或多个知识库。创建知识库并完成文件解析后，你可以开始 AI 对话。

1. 点击页面中上方的 **Chat** 选项卡 **>** **Create chat** 创建聊天助手。
2. 点击已创建的聊天应用，进入其配置页面。

   > RAGFlow 提供灵活性，允许为每个对话选择不同的聊天模型，同时你可以在 **System Model Settings** 中设置默认模型。

3. 在配置页面右侧更新 **Chat setting**：
   - 为你的助手命名并指定你的知识库。
   - **Empty response**:
     - 如果你想*限制* RAGFlow 的答案仅来自你的知识库，请在此处填写回复。当它没有检索到答案时，它将*统一*返回你设置的内容。
     - 如果你希望 RAGFlow 在无法从知识库检索到答案时*即兴发挥*，请留空，这可能会导致幻觉。

4. 更新 **System prompt** 或暂时保持默认设置。

5. 在 **Model** 下拉列表中选择聊天模型。

6. 现在，让我们开始吧：

   ![chat_thermal_solution](https://raw.githubusercontent.com/infiniflow/ragflow-docs/main/images/chat_thermal_solution.jpg)
